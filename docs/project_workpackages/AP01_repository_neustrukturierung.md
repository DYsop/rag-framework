# AP01 – Repository-Neustrukturierung

## Ziel

Das bestehende Repository in ein wissenschaftlich sauberes, reproduzierbares Forschungsartefakt überführen, dessen Struktur die Kapitel- und Forschungslogik der Masterarbeit abbildet.

## Bezug zur Masterarbeit

Grundlage für alle weiteren Arbeitspakete; entspricht der technischen Rahmensetzung (Kapitel 04 – Forschungsdesign und Architektur).

## Aufgaben

- [ ] Bestehende Inhalte sichten und im Backup-Branch sichern
- [ ] Zielstruktur (docs, src, configs, notebooks, tests, .github) anlegen
- [ ] Sinnvolle bestehende Inhalte migrieren
- [ ] .gitignore und Datenstrategie schärfen

## Akzeptanzkriterien

- [ ] Backup des Ausgangsstands existiert
- [ ] Neue Struktur ist konsistent und nachvollziehbar
- [ ] Keine vertraulichen/großen Rohdaten im Repository

## Betroffene Dateien/Ordner

`README.md`, `.gitignore`, `docs/`, `src/`, `configs/`, `tests/`, `.github/`

## Erwartetes Ergebnis

Lauffähige, dokumentierte Zielstruktur als Basis für die Folgepakete.
